Javascript variables are containers for Data with variables being declared in four different ways: Modern javascript uses let or const in older javascript var or it being done automatically.

Its no longer recomnded to use var or have the system do it automacally

JavaScript Identifiers

JavaScript identifiers are names used to identify variables, functions, and objects. They must start with a letter, underscore (_), or dollar sign ($), followed by letters, digits, underscores, or dollar signs; but be warned These are case sensitive, A CAR is different from a car.

You may not use identifiers that are Reserved by JavaScript itself, and example of this is let or class

Scopes

JavaScript has different scopes: global, function, and block scope. Global scope means a variable is accessible anywhere in the program, function scope means a variable is only accessible within a function, and block scope means a variable is only accessible within the block it was declared in.

blocked scope

blocked scope is a concept in JavaScript where variables declared with `let` or `const` are only accessible within the block they are defined in, and not outside of it.

Global scope

Global scope is a concept in JavaScript where variables declared outside of any function or block are accessible from anywhere in the program.


Declaration

Variables declared with `let` or `const` cannot be redeclared within the same scope, which helps prevent accidental overwriting of variable values.

Variables declared with `var` can be redeclared within the same scope, which can lead to unexpected behavior and bugs in your code.

JavaScript Let variable

let is a keyword used to declare variables with a block scope, variables declared with let must be declared before use and cannot be redeclared within the same scope.

JavaScript const variable

const is a keyword used to declare block scoped variables that cannot change/reassign different values
